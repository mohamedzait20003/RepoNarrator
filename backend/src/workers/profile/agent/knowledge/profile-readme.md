title: What a GitHub profile README is
---
The artifact is a GitHub *profile* README — the Markdown rendered on a
developer's profile page from the special repository whose name matches their
username (e.g. `torvalds/torvalds`). It is the first thing a recruiter, hiring
manager, or collaborator sees, and they skim it in seconds.

A strong profile README:

- Opens with a clear, specific hook — who this person is and what they build —
  not a generic greeting.
- Is scannable: short sections with headings, grouped badges, and links out to
  real work.
- Tells a small, true story: what they build, where they've worked, what they've
  shipped.
- Is written in the developer's own voice, in GitHub-Flavored Markdown.

Typical anatomy, top to bottom: optional name/tagline header → intro → skills →
work experience → featured projects → GitHub stats → optional contact/socials. A
section is included only when the sources actually support it — an honest short
README beats a padded long one.
