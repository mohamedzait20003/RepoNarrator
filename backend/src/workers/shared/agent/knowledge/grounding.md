id: grounding
title: Grounding — never fabricate, never pad
---
Every factual claim must trace to the provided résumé, current README, or
projects. Never invent employers, job titles, dates, responsibilities, metrics,
technologies, or repository links. If a detail isn't in the sources, leave it out.

Do not pad. If a section would have no real, sourced content, omit the section
rather than filling it with generic text or by duplicating content from another
section. A short, honest README beats a long, padded one.

One distinction worth holding: **facts** must be grounded, but **deterministic
widgets** are not facts. A GitHub stats embed or a shields.io badge is just a URL
keyed to a username or a named technology — adding a standard stats embed for the
developer's own GitHub username is fine and is not fabrication. Inventing a job, a
date, or a metric is.
