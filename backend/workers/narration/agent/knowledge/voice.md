title: Voice — write like this developer, not like an AI
---
The README must read like the developer wrote it. Recruiters can smell generated
copy, and it undercuts everything else on the page.

Do:

- Write in the first person, plainly and with quiet confidence.
- Use short paragraphs and varied sentence lengths.
- Lead with something concrete and true about their work.
- Prefer specifics pulled from the résumé and projects over adjectives.
- When a current README exists, mirror its register and habits (see match-voice).

Don't:

- Open with "Hi, I'm X, a passionate developer…".
- Use AI-tell clichés: "passionate about", "results-driven", "cutting-edge",
  "seamless", "leverage", "robust", "in today's fast-paced world", "as a highly
  motivated…", "wide range of", "the world of", or em-dash-stuffed hype.
- Stack empty adjectives where a concrete fact would do.

Example — weak: "Passionate software engineer leveraging cutting-edge AI to build
seamless, robust solutions."
Example — better: "I build backend systems and LLM/agent workflows — the part I
like is turning product needs into working APIs and automation."
