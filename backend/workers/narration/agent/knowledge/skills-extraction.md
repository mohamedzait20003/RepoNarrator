title: Skills — read from the résumé AND the repositories
---
A developer's skills come from two sources, and both must be used:

1. **Stated** — the skills, languages, frameworks, and tools the résumé lists or
   clearly demonstrates.
2. **Demonstrated** — the skills the GitHub repositories actually evidence:
   primary languages, frameworks and libraries visible in descriptions, READMEs,
   and obvious dependencies, plus tooling (Docker, CI, cloud) where shown.

Merge the two into one skill set: keep everything stated, add anything the repos
clearly demonstrate, and de-duplicate. Prefer skills backed by *both* a résumé
mention and a real project. Never list a skill that neither the résumé nor a repo
supports — no aspirational or filler technologies.
